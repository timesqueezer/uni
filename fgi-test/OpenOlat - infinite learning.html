    Response

    Der folgende Lückentext beschreibt das generelle Vorgehen eines Unentscheidbarkeitsbeweises per Reduktion:

    Füllen Sie alle Lücken aus mit: A, B, S, R, x oder y, wobei A und B Probleme, S und R Turingmaschinen und x und y Eingaben für Turingmaschinen sind. 

    Es soll gezeigt werden, dass Problem A unentscheidbar ist. Weiterhin ist bekannt, dass Problem B unentscheidbar ist.
    Wenn A entscheidbar wäre, dann gäbe es eine Turingmaschine R, die entscheidet.
    Es wird eine Turingmaschine S konstruiert, die eine Eingabe x für Problem B in eine Eingabe y für Problem A konvertiert, so dass x ∈ B genau dann, wenn  ∈ und die Turingmaschine benutzt, um zu entscheiden, ob y ∈ A gilt.
    Damit würde die Turingmaschine das Problem entscheiden. Das ist ein Widerspruch, da bekannt ist, dass Problem unentscheidbar ist. Es muss also die Annahme falsch sein, dass Problem entscheidbar ist.
    Daraus kann gefolgert werden, dass Problem unentscheidbar ist.
